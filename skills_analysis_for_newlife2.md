# Аналіз проєкту та рекомендації з інтеграції скілів

Цей звіт містить глибокий аналіз поточного вебсайту церкви «Нове Життя» та підбір найбільш корисних скілів (skills) із репозиторію `claude-skills-main` для покращення веброзробки, SEO, доступності та продуктивності.

---

## 1. Аналіз поточного проєкту («Нове Життя» — Борислав)

Проєкт є сучасним інформаційним та євангелізаційним вебресурсом. Основні характеристики та архітектурні рамки:

*   **Технологічний стек**: Next.js 16 (App Router), React Server Components (RSC), Tailwind CSS v4, Sanity CMS v3 (next-sanity, CDN для зображень), Sanity TypeGen для автогенерації TypeScript типів.
*   **Особливості архітектури (Invariants)**:
    *   Суворе розділення компонентів: спільні в `components/`, сторінкові компоненти локалізовані (наприклад, `app/(front)/ministries/components/`).
    *   Динамічний контент керується **тільки** через Sanity CMS.
    *   Обов'язкове використання `defineQuery()` та автогенерація типів перед збіркою (`prebuild`).
    *   Використання виключно токенів дизайну з `globals.css` (наприклад, `--color-primary`) без прямого використання сирих hex-кодів.
    *   Кирилиця захищена від пошкодження кодування (заборона PowerShell pipes/Out-File).
*   **Поточний статус**: Головна сторінка, контакти, історія, деталі та списки служінь/новин реалізовані.
*   **Найближчі цілі**:
    *   Сторінка «Команда» (team page).
    *   Медіасторінка / Онлайн-трансляції.
    *   Аудит контенту в Sanity.
    *   SEO-оптимізація в регіоні та висока швидкість роботи (Lighthouse Performance ≥ 80).

## 3. Детальний аналіз ключових скілів

### 🎯 Скіл 1: Schema Markup (`schema-markup`)
Допомагає впровадити семантичну розмітку, яка є основою для локального SEO церкви.

*   **Для головної сторінки**: Додавання JSON-LD розмітки типу `Place` та `HouseOfWorship` або `LocalBusiness`. Це дозволить Google показувати точний блок з інформацією про церкву, карту, контакти та години богослужінь прямо в результатах пошуку.
*   **Для сторінок служінь / подій**: Автоматична генерація розмітки `Event` на основі даних із Sanity CMS (наприклад, недільне служіння, молодіжне спілкування).
*   **Для новин**: Розмітка `NewsArticle` з авторами (`author`) та датами оновлення (`dateModified`), що також покращує ранжування у Google News та AI-асистентах (Google AI Overviews).

### ♿ Скіл 2: Accessibility Audit (`a11y-audit`)
Забезпечує відповідність сайту стандарту WCAG 2.2 Level A/AA.

*   **Колірний контраст**: Проєкт використовує світлу тему з акцентним зеленим кольором (`#4cbd89` / `var(--color-primary)`) та градієнтами. Скіл допоможе перевірити, чи має текст на кнопках та плашках достатній контраст щодо фону (мінімум 4.5:1 для звичайного тексту).
*   **Зручність для літніх людей**: Перевірка розміру клікабельних елементів (Touch Targets ≥ 44x44px), наявності фокус-рамок (`focus-visible`) та сумісності з екранними дикторами (screen readers) для незрячих.

### ⚡ Скіл 3: Performance Profiler (`performance-profiler`)
Ключовий інструмент для досягнення Lighthouse Performance ≥ 80.

*   **Оптимізація картинок**: Оскільки фото служінь та подій завантажуються динамічно із Sanity CMS, важливо правильно налаштувати `next/image` з використанням правильного розміру (`@sanity/image-url` з `.width()` / `.height()`) та сучасних форматів (webp/avif).
*   **Контроль завантаження скриптів**: Перевірка коректності роботи `next/dynamic` для важких компонентів (таких як TestimonialsGrid та FAQSplit, які підвантажуються нижче першого екрана).

## 4. Аналіз папок `engineering` та `engineering-team`

Репозиторій містить дві великі папки з інженерними скілами:
*   **`engineering-team`**: Містить 14 скілів, що моделюють ролі в розробці (Senior Frontend, DevOps, QA, ML, Data-scientist тощо).
*   **`engineering`**: Містить 40 розширених інструментів та архітектурних скілів (Database Designer, RAG Architect, CI/CD Builder, MCP Server Builder тощо).

Ми не пропускали ці папки, а провели ретельну фільтрацію їхнього вмісту, оскільки значна частина скілів там розрахована на корпоративні enterprise-системи, складну хмарну інфраструктуру або штучний інтелект.

### Які скіли з цих папок КОРИСНІ для вашого проєкту:
*   **`senior-frontend` та `senior-fullstack`** (`engineering-team/skills/`):
    *   *Чому*: Надають чіткі інструкції з використання React Server Components (RSC), оптимізації рендерингу та керування станом у Next.js, а також правильного використання Tailwind CSS v4.
*   **`codebase-onboarding`** (`engineering/skills/codebase-onboarding`):
    *   *Чому*: У вашому проєкті діє складна система з шести контекстних файлів (`context/architecture.md` тощо) та автогенерації типів. Цей скіл допоможе автоматично генерувати документацію та спростити введення нових розробників у проєкт.
*   **`env-secrets-manager`** (`engineering/skills/env-secrets-manager`):
    *   *Чому*: Дозволяє безпечно керувати локальними ключами доступу до Sanity CMS (зокрема токенами з доступом до запису контенту), запобігаючи випадковим витокам у репозиторій.
*   **`ci-cd-pipeline-builder`** (`engineering/skills/ci-cd-pipeline-builder`):
    *   *Чому*: Оскільки проєкт деплоїться на Vercel та має обов'язковий крок `prebuild` (де генеруються TS-типи через `sanity typegen`), цей скіл допоможе оптимізувати конфігурацію збірки.

### Додаткові ключові інженерні скіли (додано після перегляду)
*   **`karpathy-coder`** (`engineering/skills/karpathy-coder`): Для глибокого, чистого та продуманого написання коду з урахуванням edge-cases у Next.js/React.
*   **`grill-me` / `grill-with-docs`** (`engineering/skills/grill-me`): Для інтерактивних перевірок та архітектурних інтерв'ю перед ухваленням важливих рішень (наприклад, щодо структури даних Sanity).
*   **`handoff` / `code-tour`** (`engineering/skills/handoff`): Для створення якісної документації та онбордингу, пояснення складних частин системи для майбутньої підтримки.
*   **`autoresearch-agent`** (`engineering/skills/autoresearch-agent`): Для глибинного аналізу проблеми або пошуку найкращого архітектурного паттерну до початку генерації коду.
*   **`security-guidance` / `chaos-engineering`** (`engineering/skills/security-guidance`): Для базової перевірки безпеки та стабільності системи.

### Які скіли з цих папок були НАВМИСНО пропущені (і чому):
*   **Скіли для штучного інтелекту** (`senior-prompt-engineer`, `senior-ml-engineer`, `senior-computer-vision`, `rag-architect`, `agenthub`):
    *   *Причина*: На сайті церкви наразі немає функціоналу AI, обробки зображень нейромережами чи чат-ботів.
*   **Скіли для реляційних баз даних** (`database-designer`, `database-schema-designer`, `migration-architect`, `sql-database-assistant`):
    *   *Причина*: Сайт використовує безсерверну CMS Sanity.io, де вся схема описується безпосередньо у вихідному коді.
*   **Скіли для складної інфраструктури** (`kubernetes-operator`, `terraform-patterns`, `helm-chart-builder`, `docker-development`):
    *   *Причина*: Проєкт повністю обслуговується через Vercel. Робота з контейнерами або хмарною інфраструктурою як код (IaC) не потрібна.
*   **Скіли для командної взаємодії корпорацій** (`incident-commander`, `interview-system-designer`):
    *   *Причина*: Проєкт є локальним сайтом, розробка якого ведеться невеликою командою/одноосібно.

## 5. Інструкція з інтеграції та використання

Оскільки репозиторій скілів сумісний з **Antigravity**, ви можете легко встановити необхідні скіли.

### Крок 1: Конвертація та імпорт
Ви можете запустити скрипт встановлення безпосередньо з папки скілів для автоматичного копіювання у глобальну директорію Antigravity:

```powershell
# У папці c:\CodeData\sanity\claude-skills-main
./scripts/install.sh --tool antigravity
```
*Це автоматично згенерує та скопіює скіли у вашу системну папку інструменту `C:\Users\Dell\.gemini\antigravity\skills\`.*

### Крок 2: Локальне використання (CLI-скрипти)
Кожен із цих скілів містить готові скрипти на Python (без сторонніх залежностей, використовують лише stdlib), які ви можете запускати локально:

1.  **Для аудиту доступності (a11y)**:
    ```powershell
    python c:\CodeData\sanity\claude-skills-main\engineering-team\a11y-audit\skills\a11y-audit\scripts\a11y_scanner.py c:\CodeData\sanity\newlife2
    ```
2.  **Для перевірки контрастності кольорів**:
    ```powershell
    python c:\CodeData\sanity\claude-skills-main\engineering-team\a11y-audit\skills\a11y-audit\scripts\contrast_checker.py --file c:\CodeData\sanity\newlife2\app\globals.css
    ```
3.  **Для перевірки структурованої розмітки**:
    Допомагає генерувати або валідувати JSON-LD перед додаванням коду в Next.js:
    ```powershell
    python c:\CodeData\sanity\claude-skills-main\marketing-skill\skills\schema-markup\scripts\schema_validator.py c:\CodeData\sanity\newlife2\public\schema.json
    ```
