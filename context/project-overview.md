# Церква «Нове Життя» — Вебсайт

## Overview

Офіційний вебсайт церкви «Нове Життя» (м. Борислав, УЦХВЄ). Інформаційний та євангелізаційний ресурс для членів церкви та нових відвідувачів. Контент керується через Sanity CMS, деплой на Vercel.

## Goals

1. Надати повну інформацію про церкву, служіння, події та новини
2. Публікувати новини та анонси подій через CMS без участі розробника
3. Забезпечити SEO-оптимізацію для пошуку церкви в регіоні
4. Створити привабливий, сучасний, адаптивний дизайн
5. Підтримувати стабільний CI/CD через Vercel

## Core User Flow

1. Відвідувач потрапляє на головну сторінку
2. Бачить hero-слайдер, найближчі події, привітання пастора
3. Ознайомлюється зі служіннями, розкладом, новинами
4. Переходить на сторінку конкретного служіння або новини
5. Знаходить контактну інформацію / розклад богослужінь

## Features

### Головна сторінка
- Hero-слайдер з CTA-кнопками
- Slider подій (дані з Sanity)
- Секція «Хто ми» (OurVision)
- Привітання пастора
- Секція служінь (що ви знайдете)
- Розклад богослужінь (HeroGradientImage CTA)
- NewsSlider (останні новини з Sanity)
- About з статистикою
- Свідчення (TestimonialsGrid)
- FAQ (FAQSplit)
- CTA секція (ActionBoxFullWidth)

### Служіння (`/ministries`)
- Список усіх служінь — grid з карток (дані з Sanity)
- Деталі служіння (`/ministries/[slug]`): опис, лідер (reference до `teamMember` з лінком на `/team/[slug]`), галерея, цитата з Біблії, останні новини, sidebar

### Новини (`/news`)
- Список новин з пагінацією та фільтрацією за категоріями
- Sidebar: останні новини, категорії
- Деталі новини (`/news/[slug]`)

### Події / оголошення (`/events`)
- Одна модель `event` із полем `type` (подія / оголошення)
- Список з пагінацією + sidebar «інші події» (вкл. минулі), структура як у новин
- Деталі події (`/events/[slug]`) з Event JSON-LD (тільки для типу «подія»)
- На головній секція з'являється лише за наявності актуальних (дедлайн `activeUntil`/`startDate` не минув)

### Історія (`/history`)
- Timeline церкви з фото, відео, акордеонами

### Команда (`/team`)
- 5-секційна структура: Почесні (кругле фото), Рукопокладені, Групове фото (floating window), Кандидати, Відповідальні
- Категорії: ordained, responsible, candidate, honorary (одна людина — кілька категорій)
- Деталі служителя (`/team/[slug]`) з біографією, сан, кандидатство, відповідальність

### Про нас (`/about`)
- Реальна сторінка (не плейсхолдер): Хто ми + місія, наша історія (наратив + лінк на `/history`), у що ми віримо (курована добірка віровчення УЦХВЄ), цінності, «чого очікувати», керівництво (лінк на `/team`), статистика (перенесена з головної), CTA
- Контент у `lib/about-data.ts` (типізований); статистика — єдине джерело `homepage.stats` через `getHomepage()`. JSON-LD AboutPage + BreadcrumbList. **TODO:** CMS-редагування (singleton `aboutPage`) — окремий follow-up

## Scope

### In Scope
- Публічний фронтенд (SSG/SSR)
- Sanity Studio (вбудований на /studio)
- CMS-моделі: siteSettings (singleton — контакти/розклад/соцмережі/SEO), homepage (singleton — hero-слайдер/testimonials/FAQ, далі stats), ministry, news, event (події + оголошення: type/startDate/activeUntil/location/body), teamMember
- SEO-метадані для кожної сторінки
- Адаптивний дизайн (mobile → desktop)
- Автоматична генерація типів (Sanity TypeGen)

### Out of Scope
- Авторизація / реєстрація користувачів
- Онлайн-пожертви / платежі
- Чат / месенджер / форми зворотного зв'язку
- Мультимовність (тільки українська)
- Email-розсилки / newsletter
- Блог з коментарями

## Success Criteria

1. Всі сторінки рендеряться без помилок (`npm run build` проходить)
2. Весь динамічний контент управляється через Sanity Studio
3. Lighthouse Performance ≥ 80 для homepage
4. Коректна адаптивність: mobile, tablet, desktop
5. Vercel-деплой працює стабільно без помилок
6. TypeGen типи генеруються автоматично при `prebuild`
